**Node type:** Command
**Name:** UpdateChecklistItemDescription
**Actor:** [BankAdmin](http://localhost:8080/root/trade-finance/-/wikis/actors/BankAdmin)
**Target aggregate:** [ChecklistItem](http://localhost:8080/root/trade-finance/-/wikis/entities/ChecklistItem)
**Module:** BankSettings
**Sub-module:** ChecklistManagement
**Purpose:** Replace the description of an existing checklist item with a new value supplied by the BankAdmin. Only the description is mutable via this command; sequence number and status are unchanged. Optimistic concurrency is enforced via `ConcurrencyStamp` to prevent lost updates when two admins open the edit form concurrently.
**Audience:** Private
**HTTP route:** PUT /api/private/app/bank-settings/checklist-items/{id}

**Input DTO:** `UpdateChecklistItemDescriptionInput`

| Name | Type | Required | Validation | Notes |
|---|---|---|---|---|
| `Id` | `Guid` | yes | NotEmpty | Route-bound from URL path parameter |
| `ConcurrencyStamp` | `string` | yes | NotEmpty | Token from the `GetChecklistItemForEdit` response; used for optimistic concurrency check. See [OptimisticConcurrencyStrategy](http://localhost:8080/root/trade-finance/-/wikis/decisions/OptimisticConcurrencyStrategy) |
| `Description` | `string` | yes | NotEmpty; MaximumLength(500); not whitespace-only (`Must(x => x.Trim().Length > 0)`) | Uniqueness checked by domain service against all other non-deleted items in the tenant |

**Input DTO base:** plain (no audit inheritance)
**Validation:** `UpdateChecklistItemDescriptionInputValidator` (FluentValidation)
**Output DTO:** `ChecklistItemDto` (`FullAuditedEntityDto<Guid>`) — mirrors aggregate's auditing level
**Authorization:** `TradeFinancePermissions.BankSettings.ChecklistItems.Edit`

**Preconditions:**
- Caller holds permission `TradeFinancePermissions.BankSettings.ChecklistItems.Edit`. — [FRS #15 — Permissions](http://localhost:8080/root/trade-finance/-/issues/15#9-permissions)
- Target `ChecklistItem` exists and is not soft-deleted. — [FRS #15 — Commands](http://localhost:8080/root/trade-finance/-/issues/15#4-commands)
- `ConcurrencyStamp` matches the current `ConcurrencyStamp` on the stored aggregate; mismatch throws `AbpDbConcurrencyException` (HTTP 409). — [FRS #16 — Concurrency](http://localhost:8080/root/trade-finance/-/issues/16#5-concurrency)
- The new `Description` (trimmed) is not empty and does not exceed 500 characters. — [FRS #15 — Data model](http://localhost:8080/root/trade-finance/-/issues/15#5-data-model)
- No other non-deleted `ChecklistItem` in the current tenant has the same `Description` (case-insensitive). — [FRS #15 — Uniqueness](http://localhost:8080/root/trade-finance/-/issues/15#7-uniqueness)

**Postconditions:**
- `Description` on the target item is updated to the trimmed input value.
- `ConcurrencyStamp` is regenerated by ABP as part of the update.
- `LastModificationTime` and `LastModifierId` are updated by the `FullAuditedAggregateRoot` base class.
- Behavior for no-op edits (identical description re-submitted): see [IdempotentEditAuditBehavior](http://localhost:8080/root/trade-finance/-/wikis/decisions/IdempotentEditAuditBehavior) — the default adopted here is to short-circuit before persisting, so no audit record is produced for a no-op edit unless the decision is resolved otherwise.
- The response carries the full updated `ChecklistItemDto`.

**Domain events raised:** none — no cross-module consumer in this milestone.

**Side effects:** none beyond persistence and audit trail.
